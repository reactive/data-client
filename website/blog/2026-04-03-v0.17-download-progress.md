## ProgressEndpoint {#progress-endpoint}

[`ProgressEndpoint`](/rest/api/ProgressEndpoint) extends [`RestEndpoint`](/rest/api/RestEndpoint) with download progress tracking using the browser's [ReadableStream](https://developer.mozilla.org/en-US/docs/Web/API/ReadableStream) API. The `onDownloadProgress` callback fires once per chunk with `{ loaded, total, lengthComputable }`.

Use it standalone, with [`resource()`](/rest/api/resource), or per-endpoint via [`.extend()`](/rest/api/RestEndpoint#extend):

```ts
import { ProgressEndpoint, resource, Entity } from '@data-client/rest';

class FileEntity extends Entity {
  id = '';
  name = '';
  size = 0;
  static key = 'File';
}

const FileResource = resource({
  path: '/files/:id',
  schema: FileEntity,
  // highlight-next-line
  Endpoint: ProgressEndpoint,
});

// Add progress tracking to a specific endpoint
const downloadWithProgress = FileResource.get.extend({
  // highlight-start
  onDownloadProgress({ loaded, total, lengthComputable }) {
    if (lengthComputable) {
      console.log(`${Math.round((loaded / total!) * 100)}%`);
    }
  },
  // highlight-end
});
```

`lengthComputable` is `false` when `Content-Length` is missing (chunked transfer) or `Content-Encoding` is set (compressed responses), since the header reflects the compressed size while chunks are decompressed.

When `response.body` is unavailable (e.g., older `whatwg-fetch` polyfill environments), progress tracking is silently skipped and the endpoint behaves identically to `RestEndpoint`.
